①

Create a parser. In early versions of this API, the InvisibleXml class had static methods. In version 0.6.0, that was changed. It must now be instantiated and the ParserOptions to use can be specified.

②

Parse some input.

In the case of a failed parse, the InvisibleXmlDocument will also provide details about the failure.

③

Get the XML.

Several tree models can be constructed and the parse tree can be sent to any SAX document builder.
